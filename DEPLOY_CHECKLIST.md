# Вывод в сеть — чек-лист (15–30 минут)

## Шаг 0. Локальная проверка (на твоей машине)

```powershell
.\.venv\Scripts\python.exe -m pytest          # все тесты зелёные
cd frontend; npm run build; cd ..             # фронтенд собирается
git add -A; git commit -m "v3.2: оплата, чат с ИИ, прод-настройки"
git push
```

## Шаг 1. Render (рекомендуемый путь, бесплатный старт)

1. Зайди на https://render.com → New → **Blueprint**.
2. Подключи GitHub-репозиторий проекта — Render прочитает `render.yaml` и создаст сервис.
3. В Environment задай секреты (помечены `sync: false`):
   - `APP_ADMIN_PASSWORD` — пароль администратора (логин `admin` создастся при первом старте);
   - `OPENAI_API_KEY` — для ИИ-заключений и чата (можно позже);
   - `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY` — для оплаты картой (без них работает «оплата по счёту»).
4. Deploy. Через ~5 минут сервис доступен на `https://<имя>.onrender.com`.

## Шаг 2. ЮKassa (когда появится магазин)

1. В кабинете ЮKassa: создать магазин, получить `shopId` и секретный ключ → внести в Render.
2. HTTP-уведомления: URL `https://<домен>/api/payments/yookassa-webhook`, событие `payment.succeeded`.
3. `BILLING_RETURN_URL` = `https://<домен>/billing` (куда вернуть после оплаты).
4. Тестовый платёж тестовой картой ЮKassa → тариф должен активироваться автоматически.

## Шаг 3. Продажи руками (работает сразу, без ЮKassa)

1. Создать пользователя клиенту: добавить запись в `users.json` (hash через `backend.auth.hash_password`).
2. Клиент получает триал: 5 расчётов + 5 ИИ-вызовов в месяц.
3. После оплаты счёта активировать тариф:

```bash
curl -X POST https://<домен>/api/admin/users/<login>/plan \
  -H "Authorization: Bearer <токен админа>" \
  -H "Content-Type: application/json" \
  -d '{"plan": "team", "months": 1}'
```

Токен админа: `POST /api/auth/login` с логином/паролем админа.

## Шаг 4. Проверка прода

- `GET /health` → `{"status": "ok"}`
- `POST /generate-model` без токена → 403 (публичный API закрыт)
- Вход в кабинет → Новый расчёт → вердикты на главной
- Раздел «Тариф» → видны квоты и тарифы
- «Заключение ИИ» → работает при заданном ключе

## Важно знать

- **Хранилище**: users.json, проекты и usage лежат на диске инстанса. На бесплатном Render диск эфемерный — при редеплое данные сбросятся. Для продажи подключи Render Disk (платно) или перенеси users/usage в Postgres (следующий этап).
- **Rate limiter и квоты** — на один инстанс; при масштабировании заменить на Redis.
- Streamlit-кабинет (`app.py`) — внутренний инструмент, в прод не выводится.
